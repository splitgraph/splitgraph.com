To mount a MySQL database into the Splitgraph engine using [`mysql_fdw`](https://github.com/EnterpriseDB/mysql_fdw), first, add this to your `.sgconfig`'s `mount_handlers` section:

```
mysql_fdw=splitgraph.hooks.mount_handlers.mount_mysql
```

Then:

```
sgr mount mysql_fdw -c username:password@host:port \\
    -o '{"remote_schema": "remote_schema"}' schema_name
```

This will mount a MySQL schema `remote_schema` into a local schema `schema_name` on the
Splitgraph engine. Note that if your MySQL server is running on localhost and you wish to
access it over a TCP connection, you should use `127.0.0.1` instead of `localhost` as the `host`.

To snapshot the table, making it part of an actual Splitgraph image, use the [`sgr import`](../../sgr/image_management_creation/import) command.
